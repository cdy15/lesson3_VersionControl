Key Concept #1: Git

Git is a version control system used to develop software and other control tasks. First I should tell you exactly what a version control system is. A version control system is a commit tool that records changes to a file or files over time that you can go back and grab specific versions in the future. So getting into Git, Git thinks of its data like a group of snapshots of a tiny file system. Whenever you “commit” in Git it keeps track of what all your files look like at that time period. Git works in three states: committed, modified, and staged. Git also has three main sections in a project: the Git directory, the working directory, and the staging area.

Key Concept #2: Remote Repository

A remote repository is a repository the takes place on a remote server that is used and shared among one or more users. You can get to these repositories either by an HTTPS URL or an SSH URL. Git associates a remote URL with a name, the default remote is most commonly name “origin.” To add a remote, you use the command “git remote add” and match a remote URL with a name. A remote makes it extremely easy to get a collaborative approach to development, making it easier to view, fetch, and update other people’s work.

Key Concept #3: Local Repository

A local repository is a repository that resides on a local computer of a user. The difference between a local repository and a remote repository is that a remote repository is on a remote server and is shared. There are two ways to create a local repository, create a whole new repository, or you can clone a remote repository on your local computer. On a local repository you would make local changes. In Git your local repository has exactly the same features as any other repository, and that is part of what makes Git so amazing.

Key Concept #4: VCS

So I touched on VCS (Version Control Systems) earlier but now I am going to go into more depth. In a version control system there is one main repository for all of the files you have in your project. Your team member can check out each other’s files, make changes to those files, and then commit them back to the repository for anyone to see and/or edit. There has been a problem with file sharing in the past and these version control systems solve that problem. A VCS has many benefits some being files can’t be overwritten, there’s a common repository, people can work on the same files simultaneously without any trouble, and you can revert back to older versions of your work. VCS’s make working with a team way easier than ever before.

Key Concept #5: Basic Git Commands

Over the course of the reading’s and videos we watched I learned possibly the three most used commands for Git. The first being “git add.” Git add adds all of the files you have in your directory to your Git repository. “Git commit” saves the changes you have made and allows you to make your own comments making it easy for other users to see what you change you have made to your, or possibly their, files. The third and last command is “git push” which “pushes” the now saved file into the repository for all to see and edit.
